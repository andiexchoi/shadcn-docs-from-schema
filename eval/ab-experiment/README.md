# A/B experiment

Pre-registered experiment measuring whether machine-facing documentation (compact YAML + CLAUDE.md) improves the correctness of shadcn/ui code generated by an AI coding agent.

## Start here

- [`PRE_REGISTRATION.md`](PRE_REGISTRATION.md) — hypotheses, conditions, and scoring rubric, locked before any runs.
- [`RESULTS.md`](RESULTS.md) — headline findings.
- [`results/figures/`](results/figures/) — charts.

## Reproducing

- [`harness.js`](harness.js), [`harness-agentic.js`](harness-agentic.js) — run each arm.
- [`ablations/`](ablations/) — framing-philosophy and platform-guidelines ablation setups.
- [`score.js`](score.js), [`scorers/`](scorers/) — automated scoring.
- [`report.js`](report.js), [`report-ablation.js`](report-ablation.js), [`figures.py`](figures.py) — aggregate and visualize.
- [`claude-md/`](claude-md/), [`claude-md-no-framing/`](claude-md-no-framing/), [`claude-md-no-platform-guidelines/`](claude-md-no-platform-guidelines/) — the exact context files each arm received.

## Raw artifacts

Preserved for auditability, not intended as reading material:

- [`runs/`](runs/) — every generated `.tsx` from every arm (direct / agentic / ablations), 10 runs per component per condition.
- [`results/generation-log-*.json`](results/) — per-run metadata (model, prompt, latency, errors).
- [`results/scored-*.json`](results/), [`results/aggregate.json`](results/aggregate.json) — per-run scores and aggregates.
